## 2. สาระสำคัญของโครงการ และคำสำคัญ (Keywords)

**สาระสำคัญ:**
**Tam_roi** เป็นเว็บแอปพลิเคชันที่นำเสนอประสบการณ์การสำรวจประเทศไทยในรูปแบบเกม (Gamification) โดยมีวัตถุประสงค์เพื่อส่งเสริมการท่องเที่ยวเชิงวัฒนธรรมและประวัติศาสตร์ ผู้ใช้งานจะเริ่มต้นด้วยแผนที่ที่ถูกปกคลุมด้วย "หมอกแห่งสงคราม" (Fog of War) ซึ่งจะจางหายไปเมื่อผู้ใช้เดินทางไปยังพื้นที่จริงหรือทำกิจกรรมในแอปพลิเคชัน ภายในระบบประกอบด้วยฟีเจอร์การสะสมตัวละครในประวัติศาสตร์ (Historical Figures) ผ่านระบบ Quiz และการจัดอันดับผู้เล่น (Leaderboard) เพื่อสร้างแรงจูงใจในการเรียนรู้และการสำรวจ

**คำสำคัญ (Keywords):** Gamification, Fog of War, Interactive Map, Thai History, Web Application, Geospatial, Leaderboard, Historical Figures

## 3. หลักการและเหตุผล

ในปัจจุบัน การเรียนรู้ประวัติศาสตร์ไทยมักถูกจำกัดอยู่ในตำราเรียน ทำให้คนรุ่นใหม่ขาดความสนใจ คณะผู้พัฒนาจึงเล็งเห็นโอกาสในการนำเทคโนโลยีเว็บ (Web Technology) และระบบแผนที่ (Geospatial Data) มาประยุกต์ใช้เพื่อทำให้การสำรวจพื้นที่จริงเป็นเรื่องสนุก โดยใช้แนวคิด "Fog of War" จากวิดีโอเกมมาสร้างแรงกระตุ้นให้ผู้คนอยากออกไปค้นหาพื้นที่ที่ยังไม่เคยไป และเรียนรู้เรื่องราวของบุคคลสำคัญในแต่ละพื้นที่นั้นๆ เพื่อสร้างความภาคภูมิใจและคืนความมีชีวิตชีวาให้กับประวัติศาสตร์ไทย

## 4. วัตถุประสงค์

1. เพื่อพัฒนาแพลตฟอร์มการเรียนรู้ประวัติศาสตร์ไทยผ่านการสำรวจพื้นที่จริง
2. เพื่อสร้างระบบการจัดเก็บข้อมูลการท่องเที่ยวส่วนบุคคลในรูปแบบแผนที่โต้ตอบได้
3. เพื่อส่งเสริมการท่องเที่ยวในระดับท้องถิ่นผ่านการระบุตำแหน่งสถานที่สำคัญและสินค้า OTOP

## 5. ปัญหาหรือประโยชน์ที่เป็นเหตุผลให้ควรพัฒนาโปรแกรม

**ปัญหา:**
*   เยาวชนไทยขาดแรงจูงใจในการเรียนรู้ประวัติศาสตร์และวัฒนธรรม เนื่องจากรูปแบบการนำเสนอที่ไม่ดึงดูด
*   ข้อมูลสถานที่ท่องเที่ยวเชิงประวัติศาสตร์กระจัดกระจาย ไม่มีแพลตฟอร์มที่รวบรวมและนำเสนอในรูปแบบโต้ตอบได้
*   ไม่มีระบบที่เชื่อมโยงประสบการณ์การเดินทางจริงกับการเรียนรู้ประวัติศาสตร์ในรูปแบบที่สนุกสนาน

**ประโยชน์ที่คาดว่าจะได้รับ:**
*   ผู้ใช้งานมีความสนใจในการเดินทางท่องเที่ยวเชิงประวัติศาสตร์มากขึ้น
*   เกิดฐานข้อมูลการท่องเที่ยวที่แปลกใหม่ในรูปแบบการสำรวจ
*   เป็นต้นแบบแอปพลิเคชันที่นำ Gamification มาใช้กับการศึกษาและวัฒนธรรมไทยอย่างมีประสิทธิภาพ

## 6. เป้าหมายและขอบเขตของโครงการ

### กลุ่มผู้ใช้เป้าหมาย:
1. นักเรียน นักศึกษา ที่สนใจประวัติศาสตร์ไทย
2. นักท่องเที่ยวที่ต้องการความท้าทายใหม่ๆ ในการสำรวจพื้นที่
3. ผู้ใช้ทั่วไปที่ชื่นชอบการสะสมความสำเร็จ (Achievements) ในรูปแบบดิจิทัล

### ขอบเขตการทำงาน:
*   **ระบบแผนที่ (Fog of War Map):** แสดงผลพื้นที่ประเทศไทย โดยเริ่มจากสถานะล็อค (Fogged) และจะเปิดเผย (Unfogged) เมื่อผู้ใช้เช็คอิน
*   **ระบบตัวละคร (Historical Figures):** การสะสมตัวละครจากยุคสมัยต่างๆ (เช่น สมเด็จพระเจ้าตากสิน, สุนทรภู่) เมื่อผ่านเงื่อนไขในแต่ละพื้นที่
*   **ระบบอันดับ (Leaderboard):** แสดงผลคะแนน Legacy Score และจำนวนพื้นที่ที่ค้นพบ
*   **ระบบสะสมโบราณวัตถุ (Artifacts Archive):** พื้นที่สำหรับจัดเก็บไอเทมที่ได้รับจากการสำรวจ

## 7. รายละเอียดของการพัฒนา

### 7.1 เนื้อเรื่องย่อ (Story Board) และตัวอย่างผลงาน

ผู้ใช้เริ่มต้นเข้าสู่ระบบด้วย Google หรือ Email จากนั้นพบกับแผนที่ประเทศไทยที่ถูกปกคลุมด้วยหมอก (Fog of War) สีเข้ม แผนที่จะค่อยๆ เปิดเผยเมื่อผู้ใช้ทำการ "Check-in" ในเขตต่างๆ ผ่านระบบ GPS หรือการยืนยันตำแหน่ง เมื่อเข้าถึงพื้นที่ใหม่ ระบบจะเปิด Quiz หรือภารกิจเพื่อ "Capture" ตัวละครประวัติศาสตร์ระดับ S, A, หรือ C ที่เชื่อมโยงกับพื้นที่นั้น เช่น พื้นที่อยุธยา → สมเด็จพระนเรศวรมหาราช, กรุงเทพ (บางกอก) → สุนทรภู่ คะแนน Legacy Score จะสะสมและแสดงผลบน Leaderboard สาธารณะ

**ตัวอย่างหน้าจอที่จะพัฒนา:**
- หน้า Login (Google OAuth / Email)
- หน้า Map หลัก พร้อม Fog of War overlay
- หน้า Capture ตัวละคร (Quiz Modal)
- หน้า Profile แสดง % Map Discovery และ Archive
- หน้า Leaderboard

### 7.2 เทคนิคและเทคโนโลยีที่ใช้

*   **Geospatial Algorithms:** ใช้การคำนวณพิกัด GPS และการ Polygon Matching เพื่อตรวจสอบว่าผู้ใช้อยู่ในเขตใด โดยอ้างอิงข้อมูลขอบเขตจังหวัด/เขตจาก GeoJSON
*   **Fog of War Rendering:** ใช้ Leaflet.js Canvas Layer ร่วมกับ SVG Masking เพื่อสร้างเอฟเฟกต์หมอกที่ลบออกได้อย่างราบรื่น (Smooth Reveal Animation)
*   **Database Triggers & Stored Procedures:** ใช้ PostgreSQL Trigger ใน Supabase เพื่ออัปเดตคะแนน Legacy Score และ Map Completion Percentage โดยอัตโนมัติเมื่อมีการบันทึกข้อมูลใหม่
*   **Authentication (OAuth 2.0):** ใช้ Supabase Auth ที่รองรับ Google OAuth 2.0 และ Email/Password เพื่อความปลอดภัยของบัญชีผู้ใช้
*   **Gamification Logic:** ระบบ Rarity Tier (S/A/C) สำหรับตัวละครประวัติศาสตร์ โดยใช้ Algorithm การสุ่มแบบ Weighted Random ร่วมกับเงื่อนไขการปลดล็อค

### 7.4 รายละเอียดโปรแกรมที่จะพัฒนา (Software Specification)

**Input/Output Specification:**

| Input | Output |
|---|---|
| ข้อมูลพิกัด GPS ของผู้ใช้ | การลบหมอกออกจากแผนที่ในเขตที่ตรงกัน |
| คำตอบ Quiz จากผู้ใช้ | คะแนน Legacy Score และ Capture ตัวละคร |
| Email / Google Account | Session Token และโปรไฟล์ผู้ใช้ |
| การเลือกดูแผนที่ / Archive | การแสดงผลข้อมูลสถานที่และตัวละครที่เคยพบ |

**Functional Specification:**
*   **F01 — User Authentication:** ล็อกอิน/สมัครสมาชิกผ่าน Google OAuth หรือ Email
*   **F02 — Map Exploration:** แสดงผลแผนที่ Fog of War และลบหมอกเมื่อ Check-in
*   **F03 — Historical Figure Capture:** ระบบ Quiz สำหรับ Capture ตัวละครประวัติศาสตร์
*   **F04 — Archive Management:** บันทึกและแสดงผลตัวละครที่สะสมได้
*   **F05 — Leaderboard:** จัดอันดับ Legacy Score ของผู้เล่นทั้งหมด
*   **F06 — Profile Dashboard:** แสดง % Map Discovery, Archive Count และสถิติของผู้เล่น

**โครงสร้างของซอฟต์แวร์ (Design):**
```
Frontend (HTML/CSS/JS)
├── pages/
│   ├── index.html         (Login / Landing)
│   ├── map.html           (Main Map + Fog of War)
│   ├── capture.html       (Quiz / Capture Modal)
│   ├── archive.html       (Historical Figures Collection)
│   ├── leaderboard.html   (Rankings)
│   └── profile.html       (User Dashboard)
├── js/
│   ├── auth.js            (Supabase Auth)
│   ├── map.js             (Leaflet + Fog of War Logic)
│   ├── capture.js         (Quiz Logic + Rarity System)
│   └── leaderboard.js     (Score Fetching)
└── css/
    └── style.css          (Global Styles + Animations)

Backend (Supabase)
├── Tables: profiles, user_districts, captured_figures, artifacts
├── Triggers: update_legacy_score, update_map_progress
└── Auth: Google OAuth, Email/Password
```

### 7.5 ขอบเขตและข้อจำกัดของโปรแกรมที่พัฒนา

*   **ขอบเขต:** รองรับผู้ใช้ในประเทศไทยเท่านั้น โดยครอบคลุมข้อมูลเขต/จังหวัดทั้ง 77 จังหวัด
*   **ข้อจำกัดด้าน GPS:** การ Check-in ต้องใช้อุปกรณ์ที่รองรับ Geolocation API บนเบราว์เซอร์ และต้องได้รับอนุญาตการเข้าถึงตำแหน่ง
*   **ข้อจำกัดด้านฐานข้อมูล:** จำนวนตัวละครประวัติศาสตร์ในเวอร์ชันแรกจำกัดอยู่ที่ประมาณ 20–30 ตัวละคร
*   **ข้อจำกัดด้านเครือข่าย:** ต้องการการเชื่อมต่ออินเทอร์เน็ตตลอดเวลาในการใช้งาน (ไม่รองรับ Offline Mode)
*   **ไม่รองรับ:** การ Check-in ปลอม (Spoofing) จะถูกตรวจสอบในเวอร์ชันถัดไป
