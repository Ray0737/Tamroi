Findings
1. Raid — Entry point is the figure marker on the Map tab: clicking a marker for a raid_only figure calls _startRaidEncounter(figure) (js/map.js:987-994), which checks RaidModule.canStartRaid(figure) (needs ≥ figure.raid_min_players, default 2, online guild members — toasts an error if not met) then calls window.RaidModule.openRaidModal(figure) (js/map.js:993). This is gated purely by figure.raid_only === true (js/map.js:960) — raid-only figures skip the normal quiz/legendary-encounter branches entirely and can only be captured this way. RaidModule.init(figureId) itself is not called from this flow; the actually-wired entry is openRaidModal.

2. Historical Debate — Entry point is a "ประวัติศาสตร์ที่ยังถกเถียง" (Unsolved History) button dynamically injected into the figure bio modal's footer (#figure-modal, opened from Collection tab figure cards, defined app.html:552). Injection happens in js/collection.js:457-474: the button is only created if (captures.has(figureId) && window.DebateModule) AND DB.Debates.getForFigure(figureId) resolves a debate row (else nothing is added) — so it's visible only for already-captured figures that have a seeded debate. Clicking it hides the bio modal and calls DebateModule.open(figureId) (js/collection.js:471).

3. Jigsaw — Lives in the Mission tab, #coop-missions container (app.html:336), rendered by CoopModule.load() (js/coop.js:5-65). Requires the user to be in a guild (else shows "join a guild" card, js/coop.js:139-146) and a collab_missions row with type === 'jigsaw' (js/coop.js:40). If the guild leader views it before assignments exist and the guild has ≥2 members, chapters auto-assign via DB.Coop.assignJigsawChapters (js/coop.js:42-49); otherwise members see a "waiting for leader" message (js/coop.js:156-159). Once assigned, each member sees a "ส่งสรุปบท" (submit chapter summary) button in their jigsaw card (js/coop.js:186-189) whose onclick="CoopModule.postJigsawSummary(...)" calls js/coop.js:213-225, which posts via DB.Coop.postJigsawSummary. Full class summary reveals once all 3 chapters are posted (js/coop.js:153,160-172).

Raid — Map tab. Tap a marker for a Raid-only figure (special ⚔️ icon). Needs ≥2 online guild members ready, else toast error. Opens raid lobby modal directly (map.js:987-994 → RaidModule.openRaidModal).

Debate — Collection tab → tap a figure you've already captured → bio modal opens → footer shows "ประวัติศาสตร์ที่ยังถกเถียง" button, but only if that figure has a seeded debate row in DB. No debate seeded = no button (collection.js:457-474).

Jigsaw — Mission tab → "ภารกิจกลุ่ม" (guild missions) section. Needs you in a guild + a type='jigsaw' collab mission active. First member/leader to open it with ≥2 guild members auto-assigns chapters; others see "waiting for leader" until then. Each member gets a "ส่งสรุปบท" button to submit their chapter (coop.js:40-225).

All three are conditional — won't show unless prerequisites met (Raid needs raid-only figure + guild online, Debate needs captured figure + seeded debate content, Jigsaw needs active guild + jigsaw mission).
